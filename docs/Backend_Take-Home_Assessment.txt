Payment Processing System

Problem Statement
Build a multi-currency payment processing system. The system should handle 
payments between users while maintaining proper financial records.

Core Requirements
Functional

Process payments between internal users in multiple currencies (USD, EUR, 
GBP)
Process payments to external accounts in multiple currencies (USD, EUR, 
GBP)

Technology
Golang 
PostgreSQL for data persistence
Docker Compose setup

Production Considerations
You are free to include everything you deem necessary for a production-ready 
system.
We want to see your judgment on what matters most given the time constraint. 
Document what you implemented and what you would add with more time.
